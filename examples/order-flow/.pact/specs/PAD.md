# PAD — Product Spine 产品主干

## 产品目标
帮助运营在订单进入时创建一条可确认的订单记录。

## 目标用户与核心场景
| 用户/角色 | 场景 | 目标 | 关键限制 |
|-----------|------|------|----------|
| 运营 | 客户提交线下订单信息后 | 创建订单并进入详情确认 | 失败时不能丢失输入 |

## 核心业务主流程
| Step | 用户目标 | 用户动作 | 系统状态变化 | 对应功能 | 必须体验 |
|------|----------|----------|--------------|----------|----------|
| S1 | 创建订单 | 填写客户名和商品后提交 | order:draft -> order:created | create-order | 成功后进入订单详情 |

## 功能类型定义
- 主流程功能：直接推进核心业务结果。
- 辅助功能：服务某个主流程节点，但不改变主流程。
- 管理功能：配置、维护、运营、后台管理。
- 实验功能：尚未证明属于主流程的候选能力。

## 核心实体

### order
| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| id | string | 唯一标识 | 是 |
| customerName | string | 客户名称 | 是 |
| items | string[] | 订单商品 | 是 |
| status | string | 订单状态 | 是 |

状态机：draft -> created
关联：order 由运营创建。

## 体验一致性规则
- 成功后去向：订单详情页。
- 空状态：本功能不涉及列表空状态。
- 加载态：本示例为同步函数，不涉及加载态。
- 错误提示：显示可操作原因，并留在订单表单。
- 权限不足：本示例不涉及权限。
- 重复操作：本示例不处理重复提交。
- 取消 / 返回：本示例不涉及取消。

## 全局规则
- 删除策略：本示例不涉及删除。

## 权限模型
| 角色 | 数据范围 | 限制 |
|------|---------|------|
| 运营 | 当前输入订单 | 可创建 |

## 明确不做
- 不处理支付。
- 不处理订单确认。
- 不处理库存扣减。
