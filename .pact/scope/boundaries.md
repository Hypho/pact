# 边界特征清单
> /pact.pid 执行边界判断时的参照基准。

## 高风险边界（触发即暂停，等待人工决策）

| ID | 特征 | 关键词 |
|----|------|--------|
| B-H01 | 实时通信 | 实时、WebSocket、SSE、推送、即时更新 |
| B-H02 | 并发写入同一资源 | 抢购、秒杀、库存扣减、余额变更、计数器 |
| B-H03 | 金融操作 | 支付、退款、对账、余额、转账、提现 |
| B-H04 | 跨用户聚合计算 | 排行榜、全局统计、报表 |
| B-H05 | 多表事务操作 | 同时更新多张表、操作失败需回滚 |
| B-H06 | 敏感数据处理 | 身份证、银行卡、医疗、生物特征 |

**触发格式：**
```
🚨 高风险边界：[B-HXX] [特征名]
具体表现：[这个功能哪里触碰了]
框架不能保证：[具体风险]
建议措施：[可操作建议]
⏸️ 请决策：[A]继续（说明应对方案）/ [B]拆分 / [C]重新设计
```
