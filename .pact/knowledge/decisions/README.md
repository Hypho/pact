# decisions/ — 架构决策归档

存放项目开发过程中做出的重要架构决策，供后续会话快速理解"当初为什么这么做"。

**触发时机：** `/pact.ship` 归档阶段，凡涉及以下情形，记录一条决策：
- 选择了某个技术方案而放弃了另一个
- 定义了跨功能的共同约定（不在 constitution.md 中的）
- 遇到边界情况并做出了明确取舍

**命名规范：** `[YYYYMMDD]-[主题].md`
例：`20260407-auth-strategy.md`、`20260407-soft-delete.md`

---

## 记录格式

```markdown
# 决策 — [主题]
日期：[YYYY-MM-DD] | 触发功能：[功能名]

## 决策内容
[选择了什么]

## 备选方案
[放弃了什么，为什么放弃]

## 原因
[做出这个选择的具体理由：技术约束 / 产品要求 / 时间压力等]

## 影响范围
[哪些功能 / 模块受此决策约束]

## 复查条件
[什么情况下这个决策值得重新讨论，或"长期有效"]
```
