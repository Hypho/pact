# errors/ — 失败记录目录

存放命令执行失败时的摘要记录。用于识别重复失败模式，防止同类问题在后续功能中重现。

**命名规范：** `[命令]-[YYYYMMDD].md`
例：`pact.build-20260407.md`、`pact.verify-20260407.md`

**触发时机：** 任意命令执行失败，state.md 回退至上一阶段后写入。

---

## 记录格式

```markdown
# 失败记录 — [命令名] [日期]

## 基本信息
命令：/pact.[命令]
功能：[功能名]
日期：[YYYY-MM-DD]
阶段：[失败时所在阶段]

## 失败描述
[具体失败了什么，发生在哪个 Step]

## 根因分析
[为什么会失败，是契约问题 / 实现问题 / 环境问题 / AI 幻觉]

## 处理结果
[如何解决的，或暂时搁置的原因]

## 预防建议
[下次同类功能如何避免，可留空]
```

---

> 调查重复失败模式时，在 CLAUDE.md 文件装配规则「按需层」中已声明此目录，按需加载。
